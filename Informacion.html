# NOMBRE FORMULARIO DESARROLLADO EXPLICACIÓN
1 Anthony Javier Duarte Velásquez SOCIOS Este formulario se utiliza para poder llevar un buen control de todos los datos de los Socios de Negocios, al ingresar al formulario se puede observar la carga de cada uno de ellos y diferentes operaciones que podemos realizar como: insertar nuevos registros que se irán almacenando en una base de datos en la nube, también podemos editar esos registros por si se escribió algún dato erróneo, como también poder eliminar algun socio de manera permanente.
2 Héctor Enrique Bustillo Nolasco PEDIDOS El formulario de pedidos consiste en 4 procedimientos básicos, el primer procedimiento es insertar el registro de pedidos que nos hicieron, luego tenemos el siguiente procedimiento que es actualizar, este procedimiento nos permite editar información de la tabla de un solo registro útilizando también el procedimiento de traer un solo registro, el siguiente y último procedimiento es el de eliminar un registro de la tabla. Este formulario sirve en base a nuestras APIs Rest ya previamente desarrolladas
3 Dorca Ninosca Aguilar Castro FACTURA El formulario de Factura clientes se realizó con 5 funciones: la primera es CargarFacturas Permite cargar los registros, la segunda es AgregarFactura que Habilita para poder introducir los datos de una nueva factura para agregarla a la base de datos. La tercera es CargarFactura que esta utiliza el ID de las facturas para saber cuál es la que se va a actualizar. Función ActualizarFactura, esta actualiza los datos de una factura que ya existe. Por último, tenemos EliminarFactura. Con el Ajax consolidamos las Apis, introduciendo la URL, el type(GET, POST, PUT, DELETE), datatype(JSON).
